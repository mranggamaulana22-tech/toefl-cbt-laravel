# Project Requirements & Quick Setup

This file lists minimal requirements and quick setup commands to get the project running locally and to work with the API documentation.

## System requirements
- PHP: 8.3.30
- Composer: latest compatible with PHP 8.3
- MySQL: 5.7+ / MariaDB equivalent (use same version as development environment)
- Node.js: 16+ (or the version used in project; check `package.json`)
- npm or pnpm: npm included with Node
- Tooling: Laragon (Windows) recommended by the project maintainer

## Dev / Build tools
- esbuild (configured via npm scripts)
- Tailwind CSS (via npm)

## Required environment variables (minimum)
- `APP_URL` (e.g. http://localhost:8000)
- `DB_CONNECTION`, `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD`
- `MAIL_...` if email used in flows
- `QUEUE_CONNECTION` (if you run jobs locally)
- `SCRIBE_AUTH_KEY` (optional; used for example Bearer token in generated docs)

## Quick local setup
Run these commands from the project root:

```bash
composer install
cp .env.example .env
php artisan key:generate
# configure .env DB_* values
php artisan migrate --seed
npm install
npm run dev # or npm run build
php artisan serve
```

If you use Laragon, start the web server and database from the Laragon UI and update `.env` accordingly.

## API documentation files (generated by Scribe)
- OpenAPI spec: `storage/app/scribe/openapi.yaml`
- Postman collection: `storage/app/scribe/collection.json`
- Laravel docs view (served): `/docs` route (Scribe `laravel` type)
- Static HTML (if generated): `public/docs`

To regenerate docs after changes:

```bash
php artisan scribe:generate --force
```

## API requirements / conventions
- Auth: Bearer token via `Authorization: Bearer <token>` header for protected endpoints. Scribe examples use `SCRIBE_AUTH_KEY` placeholder.
- Content-Type: `application/json`
- Accept: `application/json`
- Versioning: routes under `/api/v1/*` (check `routes/api.php`)
- Error responses: JSON with `message` and `errors` fields (HTTP 4xx/5xx)

## Troubleshooting
- If migrations fail, ensure DB user has privileges and `.env` DB settings are correct.
- If assets not loading, run `npm run dev` or `npm run build`.
- To view generated docs locally, run `php artisan serve` and open `http://127.0.0.1:8000/docs`.

## Commit recommendation
We recommend committing generated OpenAPI and Postman collection to a docs/ or `storage/app/scribe` location if you want to version the API spec:

```bash
git add storage/app/scribe/openapi.yaml storage/app/scribe/collection.json
git commit -m "Add generated API docs"
```
